Questions

  1. Why should you care about how your code is formatted?
  2. How many spaces should you indent at a time?
  3. Should the code be lowercase or uppercase?
  4. What is the recommended encoding?
  5. How should you use comment to explain your code? How do you do it in HTML and in CSS?
  6. How do you validate your HTML?
  7. What is the purpose of the alt attribute?
  8. Should you separate structure (markup), presentation (styling), and behavior (scripting)? If yes, why?
  9. Should you use double or single quotation marks around attribute values?
  10. How can you validate your CSS code?
  11. When would you use meaningful ID/class names? When would you use generic ones?

Answers

  1. You sholud care because that is what makes your code more readable and easier to understand for other programmers.
  2. There should be 4 spaces.
  3. Code should be lowercased
  4. Recommended encoding is UTF-8
  5. In HTML you comment to organize your code better and to give some tips to other programmers, and in CSS you should comment the start of a new section so you can easily know which part styles which section.
  6. You validate it using HTML5 text validators.
  7. Purpose is to help search engine find the photo and to give to photo a name which explains what photo contains in the case photo is not uploaded.
  8. You should separate it because it makes your code more readable, and easier to make some changes.
  9. You should use double quotation marks.
  10. You can validate it using CSS validator.
  11. I would use meaningful names when I work on a team project so my mates could understood the code better, and generic ones I would rarely use.
